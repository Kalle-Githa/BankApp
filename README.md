# Bank App API

---

## About the project

This project is a backend banking API created as part of my school course.  
The purpose of the assignment was to build a functional ASP.NET Web API with authentication, authorization, and database integration.

The API simulates a banking system where administrators and customers have different access and functionality.

---

## Built with

- **C#**
- **ASP.NET Web API**
- **Entity Framework Core**
- **SQL Server**
- **JWT Authentication**
- **Swagger / Postman**

---

## What I worked with

In this project I focused on:
- Building a RESTful Web API in ASP.NET
- Working with authentication and authorization using JWT tokens
- Creating role-based access for **Admin** and **Customer**
- Handling bank accounts, transactions, transfers, and loans
- Connecting the API to a SQL database
- Structuring the solution using object-oriented programming principles
- Returning correct HTTP status codes and relevant responses

---

## Features

- Login system with JWT authentication
- Role-based authorization for admin and customer
- Admin can create new customers and bank accounts
- Admin can register loans for customers
- Customers can view their accounts and balances
- Customers can view transaction history
- Customers can create additional accounts
- Customers can transfer money between their own accounts
- Customers can transfer money to other customers

---

## How to run the project

1. Clone or download the repository  
2. Open the solution in Visual Studio  
3. Configure the database connection  
4. Run database migrations if needed  
5. Start the API  
6. Test endpoints with Swagger or Postman
